FreeBSD内核的上半部和下半部

发表于 2020-03-26 更新于 2020-03-27 分类于 Operation System ， FreeBSD

操作内核主要负责给用户进程提供服务，用户进程通过系统调用获取内核服务。FreeBSD内核被分为上半部和下半部。上半部内核负责处理来自用户进程的系统调用。下半部内核处理来自硬件中断。下半部内核面对中断的活动是同步的，而到了上半部内核是异步的。上半部和下半部内核通过工作队列进行通讯。

ProtoThread

发表于 2019-10-19 更新于 2020-03-25 分类于 Operation System ， Contiki

Protothread采用事件驱动的模型。事件的来源可以是中断或其他任务。Protothread之间共享运行栈，也就是说运行时栈占用的总的大小就是占用栈最大的任务的占用量。在内存特别受限的计算机系统中，这样保证实现多任务的同时也实现极低的内存占用。
